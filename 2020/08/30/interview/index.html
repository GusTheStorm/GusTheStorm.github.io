操作系统

进程和线程和通信，虚拟内存，内存管理机制；Linux内核设计与实现；

  • [x] 进程、线程、协程的区别；
    进程是对处理器、程序和文件的抽象，他是系统分配资源的基本单位，进程间有独立的地址空间。对于进程来说，有两个独立的概念：地址空间、调度/执行。线程就是将调度和执行独立出来的一个基本单位，它代表了一个执行路径。一个进程可以被设计成多个执行路径，也就是多个线程来执行。线程没有独立的地址空间，它们共享一个进程里的地址空间。由于线程需要被调度和执行，他有自己的线程标识、记录了自己的程序计数器和寄存器值。线程切换比进程切换开销更少。协程是线程的的单位。
  • [x] 同步、异步？阻塞和非阻塞？
    同步是指程序发出功能调用之后，会忙等待，完成之后再返回。异步是指调用发出之后就返回，然后在处理完成之后通知调用者。阻塞是指指令结果返回之前，当前线程会被置为阻塞态，系统继续调度其他进程。非阻塞是指处理器将继续执行发出命令的进程的后续指令，也就是没被阻塞。
  • [x] Linux内存机制；伙伴系统和slab？
    linux内核采用同时兼容32和64位系统的四级页表机制，每个页表项指向一个页框4k。频繁申请和释放不同大小的连续页框，必然导致外部碎片。为避免此情况，引入了伙伴系统。把空闲页框分为11块链表。每个块链表分别包含1，2，4…1024个连续页框块。如果要256个页框，256链表中没有，就去512中找，如果有就把512分成两个256，如果没有就去1024找，没有就错误。这里是以2为成长倍数。问题是：如果两个256其中一个里面的一个页框被占用，也无法合并；另外还有页面浪费等问题。
    slab机制：linux系统经常分配释放一些很小的对象如进程描述符等，直接用伙伴机制会导致大量内存碎片且速度很慢。这就引入了slab。当需要一个很小的内存，slab还是从伙伴系统或许一个物理页，当不需要这个内存要释放时，归还给了slab分配器，这样再需要获取时无需从伙伴系统申请，而是继续使用刚刚释放的那块内存。

    【问题1】为什么连接的时候是三次握手，关闭的时候却是四次握手？

    答：因为当Server端收到Client端的SYN连接请求报文后，可以直接发送SYN+ACK报文。其中ACK报文是用来应答的，SYN报文是用来同步的。但是关闭连接时，当Server端收到FIN报文时，很可能并不会立即关闭SOCKET，所以只能先回复一个ACK报文，告诉Client端，”你发的FIN报文我收到了”。只有等到我Server端所有的报文都发送完了，我才能发送FIN报文，因此不能一起发送。故需要四步握手。

语法：

  • [x] C和C++中struct区别？C++中struct和class的区别；
    C中struct只是数据集合，没有函数，不能有static关键字。C++中为了兼容C保留了struct关键字，但实际上struct还是像class一样对待，只是struct默认是public的。

  • [x] 什么时候会用extern “c”

    C++预处理器根据参数签名为同名函数取不同的名字（由于支持重载），C则无法区分；在C和C++混合编译时候，需要告诉编译器加了exter “C”的段按照C标准编译以保持兼容。

  • [x] 空类大小，包含成员函数？；
    空类大小一般为1。也是1。类的大小只需要计算非静态成员的大小；成员函数不占用额外内存。

  • [x] 空类编译时会做什么，有什么函数。

  • [x] 重载、重写、隐藏；
    重载是一个类里同一个方法用不同参数列表来把方法扩展成多个方法。重写（覆盖）是子类父类的同名函数的覆盖（即virtual），他们参数特征完全一致。隐藏是没把覆盖的函数声明为virtual，导致父类的函数被屏蔽。

  • [x] 多态如何实现的？虚表指针什么时候初始化？
    通过基类指针和虚函数实现；对象执行构造函数的时候，虚表指针初始化，析构函数的时候销毁。

  • [x] RAII是什么？应用呢？
    Resource Acquisition Is Initialization，资源获取就是初始化。利用的C++中构造的对象最终会被销毁这一点。因此应用就是把资源封装在类里，当类脱离了作用域被自动销毁时，资源也就被销毁了。

  • [x] 智能指针详解；share_ptr和weak_ptr；weak_ptr？可以用原生指针吗？

  • [x] 指针与引用的区别 ？
    指针是一个复合类型，它存储着另一个类型对象的地址。引用是一个别名，底层是也是类似指针一样存放它引用的变量的地址。引用保证非空，且他所存非的地址不可更改；

  • [x] delete和delete[]的区别

    delete[]如果是对象数组会逐一调用对象析构函数，如果是内置类型，两者一样。他们都能释放内存，但是如果对象数组不用delete[]去析构所有对象的话，对象占用的资源比如Socket的端口号就不会被释放，比如线程资源不被释放就一直占用内存和CPU资源。

工程问题

  • [x] 浏览器输入一个网址，得到页面这个过程发生了什么，越详细越好；

    以下每一步都需要注意缓存情况，各个环节为了提高响应都各自设置了缓存。
    主要流程分为这样几步：
    1.浏览器进行URL解析，然后进行编码。还有地址检查、安全检查、访问限制什么的。然后就检查浏览器缓存，如果有就直接加载。没有就需要联系服务器了。
    2.DNS查询(53端口)，浏览器首先检查自己的缓存，没有再去检查系统的缓存，在再去检查路由的缓存，然后ISP的缓存，最后去根域名服务器。
    3.获得IP就要开始TCP三次握手建立连接了,根据用户输入构造HTTP报文,作为TCP的数据段然后加上TCP头部构成TCP报文段,然后加上IP头部组成IP数据包.在这之后就根据情况可能用路由表或ARP查找MAC地址.然后加上链路层的帧头部等,然后就是TCP建立连接过程.
    4.服务器一层层拆开拿到http报文后解析请求,查早自己的资源地址并返回.
    5.浏览器拿到资源查看状态码,给资源做缓存,解析具体的内容然后渲染.
