多线程mutex应该mutable

发表于 2020-07-14 更新于 2020-08-24 分类于 多线程

mutable能够突破成员变量的const性质，或者在const函数中支持更改成员变量。

编写多线程程序时，如果const成员函数要对mutex加锁、解锁就会违背其const语义。这里使用mutable修饰mutex就能够实现线程安全。

四个转型动作

发表于 2020-07-14 更新于 2020-08-24 分类于 C++

转型应该尽量避免。

四个新式转型动作

我们将T(expression)或(T)expression称作旧式转型。
新式的特点是能够明显展示出转型动作，容易debug或grep。

const_cast(expression)

将对象的常量性移除的唯一手段。

dynamic_cast(expression)

安全向下转型。即决定某对象是否归属继承体系里的某个类型，可能运行时成本较高。
尽量不用。

reinterpret_cast(expression)

执行低级转型。如将int*转型为int。

static_cast(expression)

强迫隐式转换，如将non-const转换为const对象，int转换为double等，base 转换为derived等。

智能指针的使用
