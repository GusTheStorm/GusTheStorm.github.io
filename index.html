什么是底层const、顶层const？

本文介绍const限定符的基本用法、底层和顶层const的性质和作用以及使用中容易混淆的概念。

const关键字

​ 实际代码中我们常常希望某个变量的值在初始化之后便不再允许被更改，就像设置一年总是包含12个月、分配一块固定大小的内存等；C++也已经为我们提供了满足这一需求的关键字const^ 1。正因为const对象在创建后就不能再更改，所以const对象必须初始化。

从指针上运用的const说起

  1. 指向常量的指针

​ 下面的代码示例展示了const关键字与指针关系的一个方面。

​ 上面的代码中，const放在类型名int之前，因此指针本身是自由的。我们可以称其为指向常量的指针。为了容易理解，我们不妨把指针分为两层：指向（指针内存处存放的地址）和权限（解引用操作后能否作为左值）。指向常量的指针有如下特性：

  • 指向
    其指向能够改变；能够指向常量或非常量，而普通指针无法指向常量。

  • 权限

    无论所指对象本身是否是常量，都无法通过解引用修改所指对象。

  1. 常量指针

    由于指针是对象，所以const是能够用来将指针限定为常量的。即将*放在const关键字之前。这意味着指针本身是不变的，而非所指对象。

  • 指向
    其指向无法改变；

  • 权限

    能够通过解引用修改所指对象。取决于所指对象的类型。

​ 这里大家可能会想到与常量指针名称相似的常量引用。二者性质完全不同：引用本身不是对象，它只是一个别名，其本身是无法被设定为恒定不变的。我们通常说的常量引用是指对常量的引用^2。

顶层与底层const

​ 底层const表示指针所指对象是一个常量；顶层const表示指针本身是一个常量。更一般地，顶层const可以表示任意类型的常量对象；底层const[^3]则与指针和引用这类复合类型的基本部分有关。指针类型可以同时是底层const和顶层const。

​ 关于顶层或底层const，我们可以通过拷贝操作来分析。实际上跟普通类型的赋值操作类似。

  • 拷贝时两个对象必须有相同的底层const资格，或其数据类型能够转换。

    类比：我们一般不能将常量赋给非常量。

  • 顶层const对拷贝操作没有什么影响。

constexpr

​ 用constexpr修饰表达式、变量或函数旨在告诉编译器，这是一个编译时就能得出的常量表达式。

[^3]: 慎用：只有const_cast(var)能够去掉对象var的const性质

参数的传递

两种参数传递方式：传值（value）、传址（reference）

当用传值方式将对象传递给函数时，默认情形下其值将被复制一份，复制的对象是局部定义的。函数中复制的对象存放于调用函数时分配的程序栈中。

而用传址方式传递时，函数中的参数与传入的对象直接关联。

推荐的做法

  1. 应使用更高效的传址（reference）方式传入对象；当不需要修改传递过来的对象时，应该添加 const，表明我们只是避免复制操作，而不是要修改参数。
  2. 传出对象时，应使用传值（value）方式。因为当函数结束，函数栈内存将消亡，局部对象也将消亡，返回消亡对象的地址会导致错误。而传值方式返回的是对象的副本，它在函数外依然存在（大多数C++编译器会对传值返回的对象进行NRV优化操作）。

作用域

函数在参数传递过程中，应避免过于依赖定义于 file scope 内的对象。

这些局部对象不利于代码的重用。

因为我们除了理解函数如何工作，还需要去理解这些新定义对象的工作逻辑。

对此我真的是深有体会，阅读实验室程序的代码，当一个函数定义了太多局部对象，真的很难阅读和理解此函数的工作逻辑。

可以通过尽量在参数列表中添加参数，并提供默认值来减少对局部对象的依赖。

番外篇：Mutable关键字

假设我们需要设计一个类Character，此类中需要一个Private成员容器类，比如vector类：vector feature，并希望此类能够自带Iterator机制，因此我们需要额外定义一些变量来实现：指示容器的大小size、起始位置start以及下一个目标next。

Pylon API概述

pylon API 基于 GenlCam 标准。

GenlCam定义了一种相机相关数据的描述格式和一种Node节点概念（类似ROS节点），可以是事件、数据、配置等。节点实现了 GenAPI::INode 接口。

每种数据都有各自的GenApi，比如整数型，浮点型或布尔型等。

每种数据都有参数访问模式，即available、readable or writable，而各模式都有对应的判断函数，例如 GenApi::IsWritable(camera.Width)。

Pylon API提供了诸多方便的接口。包括对相机参数的配置事件、图像获取与图像处理（存储格式、读写操作等）事件等。

NULL

NULL是在标准库头文件<stddef.h>中定义的宏

它的值是一个空指针常量

C中常数0和（void*）0都是空指针常量，而C++中只有0是。

nullptr

C++中NULL扩展为常数0，这是int型。因此如果有两个重载函数，其中一个参数为int型，另一个为char*型。

当调用该函数时传递的参数是NULL，而我们的本意是调用char*类型对应的重载版本，这时会出现我们不想要的结果，调用了 int 重载版本。

而关键字nullptr解决了这种二义性，它是一种空指针常量。

Hexo 博客搭建

下载nodejs（LTS）

  1. nodejs.org 安装完成后检查node、npm版本，在Powershell或其它命令行窗口输入：

    node -v

    npm-v

  2. 使用npm安装cnpm淘宝镜像

    npm install -g cnpm –registry=https://registry.npm.taobao.org

  3. 用cnpm全局安装hexo-cli

    cnpm install -g hexo-cli

    安装完成后可用 hexo -v 查看版本验证一下安装

  4. 建立空文件夹并进入

    mkdir blog

    cd blog/

  5. hexo初始化

    hexo init

  6. 启动Hexo

    hexo s 或 hexo server

    写完一篇博客，可以用这条命令去启动预览一下

  7. 访问反馈的地址，Bingo！

常用命令

  • hexo n “文件名”

    新建的文本默认保存在blog文件夹下的source/_posts/

  • typora 文件名.md

    使用typora编辑md博客文章，使用前确保path环境变量中包含了typora文件夹

  • cd ../.. hexo clean hexo g hexo s

    退回blog、清理、生成再启动

将博客部署到Github

​ 登录Github，新建一个repository，其中地址必须是： 账号名.github.io

​ 回到命令行，安装git插件工具：

cnpm install –save hexo-deployer-git

​ 设置_config.yml

typora _config.yml

​ 打开之后看到文末Deployment

​ type: git

​ 然后在下方添加repo、branch

​ repo: 此处输入新建repository之后的地址

​ branch: master

保存退出后，使用 hexo d 命令部署到远端
