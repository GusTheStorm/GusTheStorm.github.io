Pylon API概述

pylon API 基于 GenlCam 标准。

GenlCam定义了一种相机相关数据的描述格式和一种Node节点概念（类似ROS节点），可以是事件、数据、配置等。节点实现了 GenAPI::INode 接口。

每种数据都有各自的GenApi，比如整数型，浮点型或布尔型等。

每种数据都有参数访问模式，即available、readable or writable，而各模式都有对应的判断函数，例如 GenApi::IsWritable(camera.Width)。

Pylon API提供了诸多方便的接口。包括对相机参数的配置事件、图像获取与图像处理（存储格式、读写操作等）事件等。

关于构建项目的通用设置

This section shows the most common Microsoft Visual Studio build settings for building a 32 bit application using pylon. Consult the Advanced Topics section for more information, e.g. when a different configuration is required.

Add the following include directories for Release and Debug project configurations (Configuration Properties -> C/C++ -> General -> Additional Include Directories):

$(PYLON_DEV_DIR)\include;

Add the following library directories for Release and Debug project configurations (Configuration Properties -> Linker -> General -> Additional Library Directories):

$(PYLON_DEV_DIR)\lib\Win32;

Enable runtime type info for Release and Debug project configurations (Configuration Properties -> C/C++ -> Language -> Enable Runtime Type Info).

yes

Enable C++ exceptions for Release and Debug project configurations (Configuration Properties -> C/C++ -> Code Generation -> Enable C++ Exceptions).

yes

关于配置附加依赖项

只需要在创建新项目之后，项目属性—C++中的附加依赖项中输入

$(PYLON_DEV_DIR)\include;

Pylon 基础

  • 在使用任何Pylon方法前，需要通过

    来分别初始化Pylon runtime和摧毁所有的Pylon资源。

  • CTIFactory类是传输层模块中重要的一个部分。它相对更加底层，对Device进行各种操作。例如：设备名、信息传输模式、创建和摧毁设备对象等。Device对象创建后作为参数来创建相机实例。

  • CInstantCamera类用来创建一个相机实例。通过此实例可以直接操控相机（具体接口函数参考帮助文档），包括：相机数组、缓冲区、抓取操作、配置事件、图像事件等。此外，相机实例对象有相应成员函数来直接获取设备信息。

图像操作方式

Pylon图像无法直接作为数据来操作，需要通过其提供的智能指针类 CGrabResultPtr。

此类提供了丰富的成员函数，包括对图像各种信息进行操作、对抓取过程进行判断等。

Pylon名称空间中也提供了类似opencv中的 imshow() 这样的显示函数和读写函数。
